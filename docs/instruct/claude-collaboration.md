# Claude 협업 규약

이전 세션에서 합의된 협업 패턴. 다음 세션에서도 이어지도록 기록.

---

## 1. 스타일 디버깅 협업 규칙

**Claude 는 브라우저 DevTools 를 직접 사용할 수 없음.** 렌더된 페이지, computed style, hover/active 상태 시각 확인 불가.

### 역할 분담

| 주체       | 역할                                                                    |
| ---------- | ----------------------------------------------------------------------- |
| **Claude** | `node_modules` 소스 grep, 원인 후보 나열, 코드 작성                     |
| **사용자** | 브라우저 DevTools 사용, Styles/Computed 탭 스크린샷 공유, DOM HTML 전달 |

### 원칙 — 추측 기반 시도 금지

다음을 **시도하기 전에 먼저 원인 파악**:

- `!important` 남발
- CSS `@layer` 순서 변경
- PrimeVue `ptOptions.mergeProps` 토글
- `unstyled` prop 전환

→ 모두 **원인 확인 후의 targeted 해결책**이지, 추측 기반 시도 대상 아님.

### 권장 흐름

1. 문제 보고 (사용자)
2. Claude: 소스/타입 grep 으로 구조 파악 + 원인 후보 제시
3. 필요시 사용자에게 DevTools 로 원인 검증 요청
4. Claude: 확인된 원인 기반 targeted 수정

## 2. 사용자 의사 신호 패턴

작업 착수 전 사용자 의사를 정확히 해석.

| 신호 표현                           | Claude 행동                                       |
| ----------------------------------- | ------------------------------------------------- |
| "의견만" / "생각만" / "분석만"      | 코드 작성 금지. 분석·제안만.                      |
| "~할까?" / "~어때?" / "~하면 어때?" | 탐색 질문. 구현 전 합의. 의견 제시 + 선택지 제안. |
| "근데 ~는?" / "~안 되나?"           | 의문. 답변·의견 먼저.                             |
| "어" / "좋다" / "하자" / "진행"     | 명시적 확정. 구현 진행.                           |
| "B로 가자" / "A 로 하자"            | 선택지 확정. 해당 안 구현.                        |

**혼동 시**: "구현 진행할까요, 의견만 드릴까요?" 확인 질문.

## 3. Figma 디자인 편차 허용 정책

**Figma 는 설계안이지 픽셀 단위 spec 아님.**

- 약간의 스타일 차이 허용 (색조, 여백, radius 등 subtle 차이)
- 기능·구조 정확성 > 픽셀 정확성
- PrimeVue 기본 스타일과 Figma 차이가 10% 이내면 **PrimeVue 쪽 따름** 가능 (커스터마이즈 비용 회피)
- 두 값 충돌 시 판단 기준:
  - **크게 차이 나면 사용자 확인**
  - **사소하면 그냥 구현 후 진행** (사후 지적 받으면 조정)

→ BTabs 를 PrimeVue SelectButton 기반으로 교체할 때, 버튼 간 간격 미세 차이 허용한 결정과 일치.

## 4. 축(Vavisual Variant) 고정, 값만 확장 (디자인 시스템 원칙)

공통 컴포넌트 (`B*`) API 는 **일관된 Variant** (variant × color × size) 만 가지며, 요구 확장 시 **기존 prop 의 값을 추가**한다. 새 Variant 필요 시 협의 필요.

### 적용 예

```ts
// ✅ 좋음 — 값 확장
type Variant = "filled" | "outlined" | "text" | "link"; // 'ghost' 추가
type Color = "primary" | "secondary" | "danger"; // 'success' 추가

// ❌ 나쁨 — Variant 계속 추가
interface Props {
  variant;
  color;
  size;
  tone;
  kind;
} // Variant 폭발
```

### 이점

- B\* 컴포넌트간 학습 비용 0
- 신규 B 컴포넌트 만들 때 같은 prop 구조로 시작
- 내부 map 에 entry 추가만으로 확장
